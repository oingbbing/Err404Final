# 🍔 Eatting – 배달 주문 웹 서비스

**Eatting(잇팅)**은 배달의 민족, 요기요와 같은 배달 플랫폼을 참고하여  
**고객 · 가게 · 라이더 간 실시간 주문 흐름을 구현한 웹 기반 배달 서비스**입니다.

## 📌 프로젝트 개요

- **프로젝트명** : Eatting  
- **팀명** : !Error404  
- **프로젝트 형태** : 팀 프로젝트  
- **목표**
  - 실제 배달 서비스와 유사한 사용자 경험 제공
  - 실시간 주문 처리 시스템 구현
  - 역할별(고객/가게/라이더/관리자) 기능 분리

## 🎯 기획 의도

현대 사회에서 배달 서비스 이용자는 1,000만 명을 넘어섰으며,  
보다 **편리하고 직관적인 주문 경험**에 대한 요구가 증가하고 있습니다.

Eatting은 실제 상용 배달 앱의 구조를 분석하여  
실시간 주문 처리와 사용자 중심 UI를 목표로 기획되었습니다.

## 🧩 주요 기능

### 👤 고객
- 회원가입 / 로그인
- 가게 및 메뉴 조회
- 장바구니 및 주문
- 주문 상태 실시간 확인

### 🏪 가게
- 주문 접수 및 거절
- 조리 상태 변경
- 매출 및 정산 관리
- 리뷰 관리

### 🛵 라이더
- 배달 요청 수락
- 실시간 배달 진행
- 배달 완료 처리

### 🛠 관리자
- 회원 관리
- 가게 / 라이더 관리
- 전체 주문 및 정산 관리

## 🔄 시스템 흐름 (WebSocket)

- 고객 주문 → 가게 주문 수락  
- 가게 → 라이더 배차  
- 라이더 픽업 → 배달 완료  
- 모든 과정은 **WebSocket 기반 실시간 통신**으로 처리

## 🗂 데이터베이스 구조 (ERD)

- 가게(Store)
